В рамках данной лабораторной работы было разработано клиент-серверное приложение для управления коллекцией объектов. Взаимодействие построено на базе протокола TCP: на стороне сервера используются неблокирующие сетевые каналы Java NIO, а клиент работает через стандартные потоки ввода-вывода. Сервер реализован в однопоточном режиме и разделён на логические модули: приём подключений, чтение запросов, обработка команд и отправка ответов. Все операции над коллекцией (фильтрация, преобразование, поиск, сортировка) выполнены исключительно средствами Stream API с использованием лямбда-выражений. Обмен данными осуществляется в сериализованном виде: команды и их аргументы инкапсулируются в объекты строго типизированных классов, что исключает передачу «сырых» строк. Перед отправкой клиенту элементы коллекции автоматически сортируются по имени. Клиентская часть обеспечивает чтение команд из консоли, валидацию ввода, сериализацию запросов и корректную обработку временной недоступности сервера с помощью механизма повторных подключений. Команда save удалена из клиентского интерфейса; сохранение коллекции в файл выполняется сервером только при завершении работы или по специальной серверно-доступной команде. Все этапы работы сервера (запуск, установка соединения, получение запроса, отправка ответа, сохранение данных) логируются с помощью библиотеки Log4j2.
В результате выполнения работы были получены практические навыки проектирования распределённых приложений, работы с неблокирующим вводом-выводом и сетевыми каналами в однопоточной событийно-ориентированной архитектуре. Освоены принципы объектной сериализации, построения устойчивого клиент-серверного взаимодействия по протоколу TCP, а также применение Stream API для декларативной обработки данных. Разработанное приложение полностью соответствует техническому заданию, демонстрирует корректную реакцию на сетевые ошибки и отказоустойчивость, а также может служить надёжной основой для дальнейшего изучения многопоточных серверов, протоколов удалённого вызова методов или интеграции с системами мониторинга. Исходный код полностью задокументирован в формате Javadoc с описанием назначения классов, методов, параметров и возвращаемых значений, что обеспечивает его прозрачность, лёгкость сопровождения и готовность к масштабированию.
